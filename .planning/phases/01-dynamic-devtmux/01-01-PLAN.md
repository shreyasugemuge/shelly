  <action>
Append the devtmux implementation to the end of `config/functions.zsh` (after the existing `portfind` function). Add a section header comment matching the existing style (`# -- devtmux: Dynamic dev workspace --`).

Implement these components in order:

**1. Session constant:**
```
_DEVTMUX_SESSION="dev"
```

**2. `_devtmux_persist_dir()`** — Saves DEVTMUX_DIR to `~/.zshrc.local`:
- Takes `$1` as the directory path
- Before appending, grep `~/.zshrc.local` for existing `DEVTMUX_DIR` line — skip if found (prevents duplicates per Pitfall 5 from RESEARCH.md)
- Append `export DEVTMUX_DIR="$dir"` to `~/.zshrc.local`
- Print dim gray dot: `\033[0;90m` + message

**3. `_devtmux_get_code_dir()`** — Discovers code folder:
- Check `$DEVTMUX_DIR` first, fall back to `$HOME/code`
- If directory doesn't exist: prompt user with `read -r "input?Enter path to your code folder: "`
- Expand leading `~` via `${input/#\~/$HOME}`
- If entered path doesn't exist: print red error, return 1
- Call `_devtmux_persist_dir` and `export DEVTMUX_DIR="$dir"` for entered paths
- Echo the final directory path (callers capture via `$()`)

**4. `_devtmux_pick_projects()`** — Interactive project picker:
- Takes `$1` as code directory
- Build `repos` array: iterate `"$code_dir"/*/`, skip non-`.git` dirs, collect `"name (branch)"` strings
- Get branch via `git -C "$d" rev-parse --abbrev-ref HEAD 2>/dev/null || echo '?'` (cross-platform, per RESEARCH.md)
- Get name via zsh `${d:t}` modifier (basename)
- If repos array empty: print red error "No git repos found in $code_dir", return 1
- **fzf path** (when `command -v fzf &>/dev/null`): pipe repos through `fzf --multi --prompt='Select projects (Tab=mark, Enter=confirm): ' --header='devtmux -- pick 1-3 projects'`
- **Numbered fallback**: print numbered list, `read -r "choices?Select projects (1-3, space-separated): "`, parse with `${(z)choices}`, validate each number is in range
- Enforce max 3: if more than 3 selected, take first 3 and print yellow warning "Max 3 projects -- using first 3"
- If 0 selected after all logic: print red error, return 1
- Strip branch suffix before outputting: `${s% (*}` — echo one project name per line

**5. `_devtmux_build_session()`** — Builds the tmux layout:
- Takes project names as positional args, collect into local array `projects=("$@")`
- Get `code_dir="${DEVTMUX_DIR:-$HOME/code}"`
- Get `count=${#projects[@]}`
- Create session: `tmux new-session -d -s "$_DEVTMUX_SESSION" -c "$code_dir/${projects[1]}" -x "$(tput cols)" -y "$(tput lines)"`
- Add remaining columns (zsh 1-based): `for (( i=2; i<=count; i++ ))` do `tmux split-window -h -t "$_DEVTMUX_SESSION" -c "$code_dir/${projects[$i]}"`
- Apply `tmux select-layout -t "$_DEVTMUX_SESSION" even-horizontal` ONCE after all horizontal splits, BEFORE any vertical splits (critical per Pitfall 3)
- Add 15% bottom panes per column: for each column, `tmux split-window -v -t "$_DEVTMUX_SESSION:0.$i" -p 15 -c "$code_dir/${projects[$((i+1))]}"` — NOTE: use 0-based pane index `i` (0..count-1) for tmux targeting but 1-based `i+1` for zsh array access
- Launch Claude Code in top panes (indices 0, 2, 4 = `i*2` for i in 0..count-1): `tmux send-keys -t "$_DEVTMUX_SESSION:0.$((i*2))" "clear && claude" C-m`
- Clear bottom panes (indices 1, 3, 5 = `i*2+1`): `tmux send-keys -t "$_DEVTMUX_SESSION:0.$((i*2+1))" "clear" C-m`
- Set status bar with magenta/purple accent (distinct from sysmon amber):
  - `status-style 'bg=colour235,fg=colour248'`
  - `status-left " #[fg=colour135,bold]devtmux#[fg=colour248] | #[fg=colour183]${(j:, :)projects}#[fg=colour248] "` with `status-left-length 60`
  - `status-right '#[fg=colour245]Ctrl-b d detach '`
  - `pane-border-style 'fg=colour237'`, `pane-active-border-style 'fg=colour135'`
  - `mouse on`
- Focus top-left pane: `tmux select-pane -t "$_DEVTMUX_SESSION:0.0"`

**6. `_devtmux_status()`** — Status subcommand:
- Check if "dev" session exists via `tmux has-session`
- If running: show green check + "devtmux session running", then list panes via `tmux list-panes -t "$_DEVTMUX_SESSION" -F '#{pane_index}: #{pane_current_path}'`
- If not running: show dim gray dot + "No devtmux session running"
- Show fzf availability: check `command -v fzf`, print green check or yellow dot accordingly

**7. `_devtmux_help()`** — Help subcommand:
- Print usage block matching sysmon help style
- Show: `devtmux` (launch), `devtmux kill` (teardown), `devtmux status` (info), `devtmux help` (this message)

**8. `_devtmux_launch()`** — Main launch orchestrator:
- Call `_devtmux_get_code_dir` — capture output, return 1 on failure
- Call `_devtmux_pick_projects "$code_dir"` — capture output into array (read lines), return 1 on failure
- Call `_devtmux_build_session` with project names as args
- Attach: if `$TMUX` is set, use `tmux switch-client -t "$_DEVTMUX_SESSION"`; otherwise `tmux attach-session -t "$_DEVTMUX_SESSION"`

**9. `devtmux()` — Entry point:**
- `case "${1:-}" in`
- `kill|stop)` — if session exists: kill it + green check; else dim gray "No devtmux session running"
- `status|info)` — call `_devtmux_status`
- `help|-h|--help)` — call `_devtmux_help`
- `*)` — if session exists: prompt `read -r "choice?devtmux session exists. [r]eattach or [k]ill and start fresh? "` — on k/K: kill session then `_devtmux_launch`; on anything else: switch-client or attach based on `$TMUX`; if no session: call `_devtmux_launch`
- `esac`

**Cross-platform (DEV-09):** No macOS-only tmux flags used. `git rev-parse` works on both platforms. `tput` is POSIX. No `$OSTYPE` guards needed since all commands are cross-platform.

**IMPORTANT zsh array notes (from RESEARCH.md Open Question 3):**
- Use 1-based array indexing for zsh arrays: `${projects[1]}` is first element
- For tmux pane indices (0-based), compute from the loop counter: top pane = `(i-1)*2`, bottom pane = `(i-1)*2+1` when using `for (( i=1; i<=count; i++ ))`
