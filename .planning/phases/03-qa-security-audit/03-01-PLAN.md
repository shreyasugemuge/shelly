  <name>Task 1: Install shellcheck, create .shellcheckrc, fix SC2155 violations, and add inline suppressions</name>
  <files>.shellcheckrc, config/environment.zsh, config/deps.zsh, config/functions.zsh, config/monitor.zsh, config/sysinfo.zsh, config/aliases.zsh, config/prompt.zsh</files>
  <action>
1. Install shellcheck: `brew install shellcheck` (skip if already installed via `command -v shellcheck`).

2. Create `.shellcheckrc` at repo root with MINIMAL global suppressions:
   ```
   shell=bash
   # SC2039: In POSIX sh, 'local' is undefined — these are zsh files, local is valid
   disable=SC2039
   ```
   Do NOT add broad disables like SC2086. Keep it minimal.

3. Run `shellcheck --shell=bash config/*.zsh` and capture ALL output. This is the baseline.

4. Fix REAL bugs (SC2155 — declare and assign separately):
   - config/environment.zsh: Change `export GPG_TTY=$(tty)` to `GPG_TTY=$(tty)` then `export GPG_TTY` on next line.
   - config/deps.zsh: Change `local brew_share="$(brew --prefix 2>/dev/null)/share"` to `local brew_share` then `brew_share="$(brew --prefix 2>/dev/null)/share"` on next line.

5. Add inline `# shellcheck disable=SCXXXX` directives for zsh-specific false positives:
   - Every `if $IS_MACOS` or `if $IS_LINUX` line: add `# shellcheck disable=SCXXXX` on the line ABOVE (use the exact SC code from step 3 output — likely SC2128 or SC2086). Include comment: `# IS_MACOS is a true/false boolean set in .zshrc`
   - Any zsh-specific syntax shellcheck flags (array indexing `${arr[1]}`, `typeset -U`, `(( ))`, `${var:h}` modifiers): add targeted inline disables with justification comments.
   - The `locip` alias in aliases.zsh: if shellcheck flags it, add inline disable with comment explaining the nested quoting is intentional.

6. IMPORTANT: Do NOT change the `if $IS_MACOS` idiom itself. Only add shellcheck disable comments. This is a Phase 2 design decision (see research pitfall 4).

7. Re-run `shellcheck --shell=bash config/*.zsh` after all fixes. The output must be clean (zero warnings) or every remaining warning must have an inline disable with justification.

8. Run `exec zsh` equivalent check: `zsh -c 'source .zshrc && echo "Shell loads OK"'` from the repo parent directory to confirm no syntax errors introduced. Note: this may not work in the executor environment — at minimum, verify all modified files parse: `zsh -n config/environment.zsh && zsh -n config/deps.zsh && zsh -n config/functions.zsh`.
  </action>
  <verify>
    <automated>shellcheck --shell=bash config/*.zsh 2>&1; echo "EXIT: $?"</automated>
  </verify>
  <done>shellcheck produces zero output (all real issues fixed, all zsh false positives suppressed with inline directives and justification comments). .shellcheckrc exists with shell=bash. No behavioral changes to any function — only declare/assign splits and comment additions.</done>
