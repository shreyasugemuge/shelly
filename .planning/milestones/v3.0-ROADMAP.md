# Milestone v3.0: Dynamic Dev Workspace + Quality Hardening

**Status:** SHIPPED 2026-03-14
**Phases:** 1-3
**Total Plans:** 5

## Overview

v3.0 transforms Shelly from a shell configuration into a development workspace platform. The dynamic `devtmux` command replaces a hardcoded alias with an interactive multi-project tmux launcher. Tech debt cleanup centralizes platform detection and deduplicates PATH. QA hardening establishes a shellcheck zero-warning baseline, adds input validation, and audits for secrets.

## Phases

### Phase 1: Dynamic devtmux

**Goal:** Replace hardcoded `devtmux` alias with a dynamic function that lets users pick 1-3 projects and opens a tmux workspace with Claude Code + terminal for each.
**Depends on:** Nothing
**Plans:** 2 plans

- [x] 01-01: Implement complete devtmux function in functions.zsh (picker, layout, session mgmt, subcommands)
- [x] 01-02: Remove old alias from aliases.zsh and human-verify end-to-end workflow

**Key deliverables:**
- `devtmux` function in `config/functions.zsh`
- Code folder detection (`~/code` default, prompt if missing)
- Project picker (fzf with numbered-list fallback)
- tmux layout: N columns, Claude Code top + terminal bottom per column
- Session management (reattach / kill+recreate / `devtmux kill`)
- Cross-platform (macOS + Linux)

---

### Phase 2: Tech Debt Cleanup

**Goal:** Reduce duplication and fragility by centralizing platform detection, deduplicating PATH, and removing hardcoded paths.
**Depends on:** Phase 1
**Plans:** 1 plan

- [x] 02-01: Centralize platform detection, PATH dedup, and doc cleanup

**Key deliverables:**
- `IS_MACOS` / `IS_LINUX` variables set once, used everywhere
- PATH dedup via `typeset -U path` in environment.zsh
- 11 scattered `$OSTYPE` checks replaced

### Phase 3: QA & Security Audit

**Goal:** Harden the codebase with linting, input validation, and security review.
**Depends on:** Phase 2
**Plans:** 2 plans

- [x] 03-01: Shellcheck audit, SC2155 fixes, inline suppressions, and input validation for portfind/mkcd
- [x] 03-02: Plugin sourcing order guard and secrets/paths audit with human verification

**Key deliverables:**
- shellcheck zero-warning baseline with documented inline suppressions
- Input validation for portfind (port range 1-65535) and mkcd (empty args)

## Milestone Summary

**Key Decisions:**
- Used 1-based zsh loop with (i-1)*2 pane index formula for tmux pane targeting
- stderr for errors in helpers so stdout captures only project names/paths
- fzf optional: command -v guard with numbered fallback for cross-platform UX
- IS_MACOS/IS_LINUX set once in .zshrc (no export), used as booleans in all 5 config modules
- typeset -U path deduplicates PATH preserving first-occurrence order
- Guard comment block co-located with zsh-syntax-highlighting source line

**Issues Resolved:**
- 11 scattered $OSTYPE checks consolidated to single assignment
- SC2155 violations fixed (declaration/assignment separation)
- Plugin sourcing order documented with guard comment

**Issues Deferred:**
- None

**Technical Debt Incurred:**
- None identified
